[Step 1: 프로젝트 초기 설정 및 공통 레이아웃/라우팅 구축]**을 위해, Next.js App Router와 Tailwind CSS 기반의 가장 견고하고 깔끔한 기초 뼈대 설계를 완료했습니다.

이번 단계에서는 전체적인 페이지의 '틀'과 '흐름'을 잡는 데 집중했습니다:

Step 1: AppShell & Home Skeleton: 고정된 상단 헤더와 중앙 정렬된 메인 컨테이너, 그리고 하단 푸터로 구성된 공통 레이아웃입니다. 헤더는 스크롤 시 블러 효과와 미세한 경계선을 제공하며, 현재 위치를 직관적으로 알 수 있는 활성 메뉴 표시를 포함합니다.
About, 포트폴리오, Projects, Blog, Scrap, Contact 페이지 스켈레톤: 각 라우트에 접근했을 때 메인 콘텐츠가 정상적으로 교체되는지 확인할 수 있도록 기초 H1 태그가 배치된 뼈대 페이지들입니다.
관리자 페이지(Studio) 기초: 공개 메뉴에는 노출되지 않지만, /studio 경로를 통해 접근 가능한 대시보드 및 작성 페이지의 라우팅 구조를 확보했습니다.
이 모든 구조는 Apple/Notion 스타일의 Light mode 모던 미니멀리즘 톤을 엄격히 준수하며 설계되었습니다.


 AppShell과 라우팅 뼈대를 바탕으로, 이번에는 **[Step 2: 주요 퍼블릭 페이지(Home, About, Contact) 상세 구현]**을 진행할 거야. 기획서의 6.1, 6.2, 6.6 항목을 참고해서 아래 요구사항에 맞게 각 페이지의 UI 컴포넌트를 작성해 줘. 아직 백엔드 데이터가 없으므로, 레이아웃을 확인할 수 있도록 "Backend & AI Agent 프리랜서 개발자" 콘셉트에 맞는 그럴듯한 더미 데이터(Mock data)를 하드코딩해서 채워 넣어 줘.

[Step 2 요구사항]
1. Home (/) 페이지 구현

Hero 섹션: 메인 카피 "Backend & AI Agent Freelance Engineer", 보조 카피(운영/자동화/에이전트/엔터프라이즈 해결), CTA 버튼 2개(프로젝트 보기(Primary), 연락하기(Secondary)) 배치.
Highlights 섹션: 운영/프로덕션, 설계/개발, 자동화/에이전트 등의 강점을 나타내는 3~4개의 심플한 카드 UI.
Featured 포트폴리오 & Latest Updates: 대표 프로젝트 3개를 보여주는 카드 리스트(더 보기 링크 포함)와 최신 블로그/스크랩 항목을 보여주는 섹션. 2. About (/about) 페이지 구현
프로필 요약(3~5줄) 및 주요 태그.
Resume 섹션: 타임라인이나 리스트 형태로 경력과 주요 성과(수치 기반)를 스캔하기 쉽게 배치.
Skills 섹션: Backend, Infra, AI, Tooling 등 카테고리별로 기술 스택 정리.
화면 하단에 Resume PDF 다운로드 및 Contact로 넘어가는 CTA 배치. 3. Contact (/contact) 페이지 구현
이메일 주소 노출 및 소셜 링크(GitHub, LinkedIn).
(선택 사항) 간단한 문의 폼 UI.
안내 문구: "일하는 방식"(의뢰 범위, 커뮤니케이션 방식, 프로세스 등)에 대한 짧은 가이드 텍스트 배치. 디자인 가이드 (반드시 지킬 것)
Apple이나 Notion 스타일의 모던 미니멀리즘 유지.
과한 색상은 빼고, 여백, 타이포그래피(16px 본문, 24/32/40px 제목 스케일), 그리고 얇은 Border 라인과 카드로만 구분할 것.
컴포넌트는 재사용 가능하게 분리해서 작성해 줘 (예: 포트폴리오Card, HighlightCard).

이제 구조가 잡혔으니, **[Step 3: 데이터/마크다운 기반 페이지(포트폴리오, Blog) 상세 구현]**을 진행할 거야. 기획서의 6.3과 6.4 항목을 바탕으로, 아래 추가된 핵심 요구사항을 반드시 반영해서 UI와 인터랙션을 구현해 줘. 아직 실제 DB가 연결되지 않았으므로, 상태 관리(State)와 더미 데이터(Mock data)를 활용해서 실제 작동하는 것처럼 만들어야 해.

[Step 3 요구사항]
1. 포트폴리오 (/portfolio) 페이지 구현

카드 리스트 UI: 그리드 형태(데스크탑 기준 2~3열)의 카드 컴포넌트(포트폴리오Card)를 만들어 줘.
카드 필수 포함 요소:
상단 썸네일 이미지 (Unsplash 등의 더미 이미지 URL 적용)
프로젝트 제목 및 한 줄 설명
기술 컨셉 포인트: 단순 태그가 아니라, 해당 프로젝트의 핵심 기술이나 아키텍처 포인트를 2~3개의 Bullet 형태로 보여줄 것.
인터랙션: 카드 전체 영역을 클릭하면 해당 프로젝트의 상세 페이지(/portfolio/[slug])로 부드럽게 라우팅되도록 Link 컴포넌트로 감싸 줘.
더미 데이터 컨셉: 'LLM 기반 멀티 에이전트 서비스(회의실 예약, 뉴스 큐레이션, AI 보고서 자동 생성)', 'FastAPI와 React를 연동한 비동기 백엔드 아키텍처' 등 실제 백엔드/AI 에이전트 개발 포트폴리오에 맞는 데이터를 3~4개 하드코딩해 줘. 

2. Projects (/projects) 페이지 구현
회사를 다니면서 진행했던 시스템 프로젝트 이력(히스토리)를 연대기(Timeline) 순으로 나열하는 페이지 구현. HTML 템플릿의 타임라인 UI 구조를 React 컴포넌트로 변환.

3. Blog (/blog) 페이지 구현
'학습(Learning)'이라는 목적에 맞게, 화면 좌측(또는 상단)에 트리 구조(Tree-structure)의 카테고리 필터링 UI를 구현해 줘.
트리 구조 세부 요건:
폴더처럼 접기/펴기(Collapse/Expand)가 가능한 아코디언 스타일.
예시 트리 데이터:
Backend (하위: FastAPI, Python Architecture)
AI & Agent (하위: LLM, Prompt Engineering)
Frontend (하위: React)
메인 목록 영역: 트리에서 선택한 카테고리에 맞춰 우측의 블로그 글 목록(제목, 요약, 작성일, 읽는 시간)이 필터링되어 렌더링되도록 상태(State) 관리를 구현해 줘. 디자인 및 컴포넌트 가이드
모던 미니멀리즘 원칙을 유지하며, 카드에 마우스를 올렸을 때(Hover) 미세한 그림자(Shadow)나 스케일(Scale) 변화를 주어 클릭을 유도할 것.
트리 필터링 영역과 메인 글 목록 영역 사이의 여백(Gap/Padding)을 넉넉히 주어 가독성을 높일 것.

론트엔드 UI의 마지막 단계인 **[Step 4: 관리자용 Studio 대시보드 및 에디터 구현]**을 진행할 차례야. 기획서의 6.7 항목을 바탕으로, 나만의 콘텐츠(블로그, 스크랩)를 생산하고 관리하는 Private 영역(/studio)을 구축해 줘. 철저하게 '작업 생산성'에 초점을 맞춘 UI여야 해.

[Step 4 요구사항]
1. Studio 대시보드 (/studio)

화면 상단에 Draft(초안)와 Published(발행됨) 상태를 전환할 수 있는 탭 UI 구현.
작성된 글/스크랩의 리스트(제목, 타입, 업데이트 날짜, 태그, 상태)를 한눈에 볼 수 있는 깔끔한 테이블 또는 리스트 뷰.
눈에 띄는 곳에 '새 블로그 글', '새 스크랩' 작성 액션 버튼 배치. 2. 마크다운 에디터 (/studio/blog/new, /studio/scrap/new)
Split View 레이아웃: 좌측은 마크다운 문법을 입력하는 텍스트 에디터 영역, 우측은 실시간으로 렌더링되는 프리뷰 영역으로 정확히 반분(50:50)할 것. (VSCode의 마크다운 미리보기 경험과 유사하게 구성).
우측 또는 숨김 가능한 사이드바 (메타데이터): 제목, Slug, 요약, 태그, 공개 여부(Public/Private)를 입력하는 폼 컴포넌트 배치.
상단 고정 액션 바: '← 목록으로', '저장(Draft)', '발행(Publish)', '내보내기(Export)' 버튼 구성. 3. 인터랙션 및 상태(State) 연결
에디터 텍스트 영역에 글을 입력하면 React의 상태(State)가 업데이트되어 우측 프리뷰 영역에 즉각적으로 마크다운이 렌더링 되도록 연결해 줘.
Export 기능 (Mock): '내보내기' 버튼 클릭 시, 프론트엔드 단에서 현재 작성된 메타데이터와 본문을 조합하여 .md 파일로 다운로드하는 로직(Blob API 활용)을 간단한 Mock 함수로 구현해 둘 것.
전체적인 룩앤필은 Notion처럼 여백이 넉넉하고 텍스트에만 집중할 수 있는 미니멀한 환경으로 만들어 줘.
