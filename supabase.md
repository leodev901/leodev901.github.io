# Supabase 설계 메모

## 1. 전제

- 이 프로젝트는 `GitHub Pages`에 올라가는 **정적 페이지** 기준입니다.
- 따라서 `/studio`는 **Next.js 서버 액션 / 서버 API / `cookies()` 기반 SSR 인증**에 의존하면 안 됩니다.
- `/studio`는 브라우저에서 `supabase-js` 또는 `@supabase/ssr`의 **browser client**로 직접 Supabase에 접근하는 방식으로 동작해야 합니다.
- 정적 페이지에서 직접 DB를 호출하므로, **RLS(Row Level Security)** 정책이 반드시 필요합니다.

## 4. 정적 GitHub Pages에서의 동작 원칙

`/studio`를 실제로 동작시키려면 아래 원칙으로 가야 합니다.

1. `/studio` 화면은 가능하면 `use client` 컴포넌트로 전환
2. 로그인은 브라우저에서 `supabase.auth.signInWithPassword()` 사용
3. 목록/작성/수정/발행은 브라우저에서 직접 `supabase.from(...)` 호출
4. anon key가 브라우저에 노출되므로, **RLS 정책으로 작성자 본인 또는 admin만 쓰기 허용**
5. 공개 페이지(`/blog`, `/scrap`)는 **로그인 없이도 읽기 가능**해야 함

즉, 정적 페이지여도 **Supabase를 BaaS(Backend as a Service)** 로 직접 붙이면 `/studio`는 충분히 운영 가능합니다.
읽기는 공개하고, 쓰기만 작성자 본인 또는 admin으로 제한하는 방식입니다.

## 5. 실행용 DDL

아래 SQL은 Supabase SQL Editor에서 순서대로 실행하면 됩니다.

```sql
-- ============================================================
-- 0) 기본 준비
-- ============================================================
create schema if not exists leodev901;
create schema if not exists private;

create extension if not exists pgcrypto;

-- ============================================================
-- 1) ENUM 타입
-- ============================================================
do $$
begin
  if not exists (
    select 1
    from pg_type t
    join pg_namespace n on n.oid = t.typnamespace
    where t.typname = 'content_status'
      and n.nspname = 'leodev901'
  ) then
    create type leodev901.content_status as enum ('draft', 'published', 'archived');
  end if;
end
$$;

do $$
begin
  if not exists (
    select 1
    from pg_type t
    join pg_namespace n on n.oid = t.typnamespace
    where t.typname = 'content_visibility'
      and n.nspname = 'leodev901'
  ) then
    create type leodev901.content_visibility as enum ('public', 'private');
  end if;
end
$$;

do $$
begin
  if not exists (
    select 1
    from pg_type t
    join pg_namespace n on n.oid = t.typnamespace
    where t.typname = 'app_role'
      and n.nspname = 'leodev901'
  ) then
    create type leodev901.app_role as enum ('admin', 'user');
  end if;
end
$$;

-- ============================================================
-- 2) 공통 함수
-- ============================================================
create or replace function leodev901.set_updated_at()
returns trigger
language plpgsql
as $$
begin
  new.updated_at = now();
  return new;
end;
$$;

create or replace function private.has_any_role(required_roles text[])
returns boolean
language sql
stable
security definer
set search_path = public, leodev901, private
as $$
  select exists (
    select 1
    from leodev901.user_roles ur
    where ur.user_id = auth.uid()
      and ur.role::text = any(required_roles)
  );
$$;

-- ============================================================
-- 3) 역할 테이블
-- ============================================================
create table if not exists leodev901.user_roles (
  id bigint generated by default as identity primary key,
  user_id uuid not null references auth.users(id) on delete cascade,
  role leodev901.app_role not null default 'user',
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint uq_user_roles_user_id unique (user_id)
);

drop trigger if exists trg_user_roles_updated_at on leodev901.user_roles;
create trigger trg_user_roles_updated_at
before update on leodev901.user_roles
for each row
execute function leodev901.set_updated_at();

-- ============================================================
-- 4) 블로그 포스트
-- ============================================================
create table if not exists leodev901.blog_posts (
  id uuid primary key default gen_random_uuid(),
  author_user_id uuid not null references auth.users(id) on delete restrict,
  slug text not null,
  title text not null,
  summary text not null,
  content_md text not null default '',
  tags text[] not null default '{}',
  cover_image_url text,
  canonical_url text,
  reading_time_minutes integer not null default 1 check (reading_time_minutes > 0),
  pinned boolean not null default false,
  status leodev901.content_status not null default 'draft',
  visibility leodev901.content_visibility not null default 'public',
  published_at timestamptz,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint uq_blog_posts_slug unique (slug),
  constraint chk_blog_posts_slug_format check (slug ~ '^[a-z0-9]+(?:-[a-z0-9]+)*$'),
  constraint chk_blog_posts_published_at
    check (
      (status = 'published' and published_at is not null)
      or
      (status <> 'published')
    )
);

create index if not exists idx_blog_posts_status_visibility_published_at
  on leodev901.blog_posts (status, visibility, published_at desc nulls last);

create index if not exists idx_blog_posts_author
  on leodev901.blog_posts (author_user_id);

create index if not exists idx_blog_posts_tags_gin
  on leodev901.blog_posts using gin (tags);

drop trigger if exists trg_blog_posts_updated_at on leodev901.blog_posts;
create trigger trg_blog_posts_updated_at
before update on leodev901.blog_posts
for each row
execute function leodev901.set_updated_at();

-- ============================================================
-- 5) 스크랩
-- ============================================================
create table if not exists leodev901.scraps (
  id uuid primary key default gen_random_uuid(),
  author_user_id uuid not null references auth.users(id) on delete restrict,
  slug text not null,
  url text not null,
  title text,
  source text not null,
  one_line_summary text not null,
  comment_md text,
  tags text[] not null default '{}',
  scrap_date date not null default current_date,
  og_title text,
  og_image_url text,
  status leodev901.content_status not null default 'draft',
  visibility leodev901.content_visibility not null default 'public',
  published_at timestamptz,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint uq_scraps_slug unique (slug),
  constraint chk_scraps_slug_format check (slug ~ '^[a-z0-9]+(?:-[a-z0-9]+)*$'),
  constraint chk_scraps_published_at
    check (
      (status = 'published' and published_at is not null)
      or
      (status <> 'published')
    )
);

create index if not exists idx_scraps_status_visibility_published_at
  on leodev901.scraps (status, visibility, published_at desc nulls last);

create index if not exists idx_scraps_author
  on leodev901.scraps (author_user_id);

create index if not exists idx_scraps_tags_gin
  on leodev901.scraps using gin (tags);

drop trigger if exists trg_scraps_updated_at on leodev901.scraps;
create trigger trg_scraps_updated_at
before update on leodev901.scraps
for each row
execute function leodev901.set_updated_at();

-- ============================================================
-- 6) 스튜디오 대시보드용 통합 뷰
-- ============================================================
create or replace view leodev901.studio_contents
with (security_invoker = true) as
select
  bp.id,
  'blog'::text as content_type,
  bp.slug,
  bp.title,
  bp.summary as preview_text,
  bp.tags,
  bp.status,
  bp.visibility,
  bp.published_at,
  bp.updated_at,
  bp.created_at
from leodev901.blog_posts bp

union all

select
  s.id,
  'scrap'::text as content_type,
  s.slug,
  coalesce(s.title, s.source) as title,
  s.one_line_summary as preview_text,
  s.tags,
  s.status,
  s.visibility,
  s.published_at,
  s.updated_at,
  s.created_at
from leodev901.scraps s;

-- ============================================================
-- 7) 실습 호환용 posts 테이블 (선택)
--    learning/blog, learning/board2, learning/realtimesub 용도
-- ============================================================
create table if not exists leodev901.posts (
  id bigint generated always as identity primary key,
  title text not null,
  summary text,
  content text not null,
  image_url text,
  created_at timestamptz not null default now()
);

create index if not exists idx_posts_created_at
  on leodev901.posts (created_at desc);
```

## 6. Storage 권장안

에디터에서 커버 이미지나 스크랩 썸네일을 올릴 가능성이 높기 때문에,
테이블 외에 Storage bucket도 하나 권장합니다.

예시:

```sql
insert into storage.buckets (id, name, public)
values ('studio-assets', 'studio-assets', true)
on conflict (id) do nothing;
```

권장 사용 방식:

- 블로그 커버 이미지
- 스크랩 OG 대체 이미지
- 에디터 내 첨부 이미지

## 7. 왜 이렇게 나눴는가

### `blog_posts`

- `PROJECT.md`의 BlogPost 모델과 거의 1:1 대응됩니다.
- 향후 `/blog/[slug]` 상세 라우트와 연결하기 쉽습니다.
- 권한 모델은 `읽기 공개 / 수정은 작성자 본인 또는 admin` 기준입니다.

### `scraps`

- 스크랩은 URL, 출처, 한 줄 요약, 내 코멘트가 핵심이라 블로그와 구조가 다릅니다.
- 블로그 테이블에 억지로 섞기보다 별도 테이블이 유지보수에 유리합니다.
- 권한 모델은 `읽기 공개 / 수정은 작성자 본인 또는 admin` 기준입니다.

### `studio_contents` 뷰

- 대시보드에서는 "All Content" 목록이 필요합니다.
- 블로그/스크랩을 각각 따로 조회한 뒤 프론트에서 합칠 수도 있지만,
  뷰 하나로 정리하면 클라이언트 코드가 더 단순해집니다.
- 단, 브라우저에서 직접 접근하는 뷰는 `security_invoker = true`로 만들어야
  기저 테이블의 RLS 정책을 그대로 따르게 할 수 있습니다.

### `private.has_any_role()` 함수

- 역할 체크 함수는 `security definer`로 동작하므로,
  공개 API 스키마인 `leodev901`가 아니라 `private` 스키마에 두는 편이 안전합니다.

## 9. 중요한 주의점

- 현재 [src/app/lib/supabase-server.ts](./src/app/lib/supabase-server.ts)는 서버 환경 전용입니다.
- `GitHub Pages` 정적 배포에서는 이 경로에 의존하는 실시간 운영 로직을 넣지 않는 것이 안전합니다.
- `/studio` 실운영은 **브라우저 클라이언트 + RLS** 조합으로 가는 것이 맞습니다.
- 즉, "정적 페이지지만 Supabase를 통해 실제 동작"은 가능하지만,
  그 전제는 **서버 없는 구조에 맞는 클라이언트 중심 구현**입니다.
